# 15. 3Sum

## Intuition

A brute-force solution checks every possible triplet, resulting in **O(n³)** time.

A better approach is to first **sort the array**. Then, fix one element and use the **Two Pointer** technique to find the remaining two numbers whose sum equals the negative of the fixed element.

Sorting also makes it easy to skip duplicate values, ensuring only unique triplets are returned.

---

## Approach

### Sorting + Two Pointers (Optimal ⭐⭐⭐)

1. Sort the array.
2. Traverse the array and fix one element `nums[i]`.
3. Skip duplicate values of `nums[i]`.
4. Initialize:
   ```
   left = i + 1
   right = n - 1
   ```
5. Calculate:
   ```
   sum = nums[i] + nums[left] + nums[right]
   ```
6. If:
   - `sum == 0`
     - Add the triplet to the answer.
     - Skip duplicate values of `left` and `right`.
     - Move both pointers.
   - `sum < 0`
     - Move `left++` to increase the sum.
   - `sum > 0`
     - Move `right--` to decrease the sum.
7. Continue until `left >= right`.

---

## Complexity

- **Time Complexity:** `O(n²)`
- **Space Complexity:** `O(1)` *(excluding the output list)*

## Approach 1: Brute Force

### Idea

Check every possible triplet and return those whose sum equals zero.

### Algorithm

1. Use three nested loops.
2. Generate every triplet `(i, j, k)`.
3. If:
   ```
   nums[i] + nums[j] + nums[k] == 0
   ```
   store the triplet.
4. Remove duplicate triplets before returning.

### Complexity

**Time:** `O(n³)`

**Space:** `O(1)`

---

## Approach 2: Sorting + HashSet (Better ⭐⭐)

### Idea

Sort the array and fix one element.

Use a HashSet to find the third element required to make the sum zero while avoiding duplicate triplets.

### Algorithm

1. Sort the array.
2. Fix one element.
3. Create an empty HashSet.
4. Traverse the remaining elements.
5. Compute:
   ```
   third = -(nums[i] + nums[j])
   ```
6. If `third` exists in the HashSet:
   - Store the triplet.
7. Otherwise:
   - Insert the current element into the HashSet.

### Complexity

**Time:** `O(n²)`

**Space:** `O(n)`

---

## Approach 3: Sorting + Two Pointers (Optimal ⭐⭐⭐)

### Idea

After sorting the array, fix one element and use two pointers to efficiently search for the remaining two numbers.

Duplicate values are skipped to ensure every triplet is unique.

### Algorithm

1. Sort the array.
2. Traverse the array.
3. Skip duplicate first elements.
4. Set:
   ```
   left = i + 1
   right = n - 1
   ```
5. While `left < right`:
   - Calculate:
     ```
     sum = nums[i] + nums[left] + nums[right]
     ```
   - If `sum == 0`:
     - Store the triplet.
     - Skip duplicate `left` and `right` values.
     - Move both pointers.
   - If `sum < 0`:
     - Move `left++`.
   - Otherwise:
     - Move `right--`.

### Complexity

**Time:** `O(n²)`

**Space:** `O(1)` *(excluding output list)*

## Formula

```
sum = nums[i] + nums[left] + nums[right]
```

If:

```
sum == 0
```

then:

```
(nums[i], nums[left], nums[right])
```

is a valid triplet.

---

## Why Skip Duplicates?

Duplicate values can produce the same triplet multiple times.

```java
if (i > 0 && nums[i] == nums[i - 1])
    continue;
```

Skips duplicate first elements.

```java
while (left < right && nums[left] == nums[left + 1])
    left++;
```

Skips duplicate left values.

```java
while (left < right && nums[right] == nums[right - 1])
    right--;
```

Skips duplicate right values.

This guarantees that every triplet appears only once.

✔ Sorting + HashSet → `O(n²)`, `O(n)`

✔ Sorting + Two Pointers → `O(n²)`, `O(1)` ⭐⭐⭐
